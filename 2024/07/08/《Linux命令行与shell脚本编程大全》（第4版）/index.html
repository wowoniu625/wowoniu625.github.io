第7章 理解Linux权限

UID

7.1 Linux的安全性

7.1.1 /etc/passwd文件

是一个专门的文件用来匹配登录名和UID值

root用户是Linux系统的管理员，其UID值为0。

系统账户：为各种各样的功能创建的账户，是系统运行中的各种服务进程访问资源使用的特殊账户，所有运行在后台的服务都需要通过一个系统账户登录到Linux系统中。Linux系统为系统账户预留了500以下的UID。

普通账户：为普通账户创建账户时，大部分Linux会从500开始将第一个可用UID分配给账户。（Ubuntu从1000开始）

该文件有7个字段。不建议使用文本编辑器直接编辑passwd文件，可能导致文件受损，应使用Linux用户管理工具。

  1. 登录用户名
  2. 用户密码
  3. UID
  4. 组ID：主要组的ID
  5. 用户账户的文本描述（备注字段）
  6. 用户家目录$HOME目录的位置
  7. 用户默认的shell

文件中的密码字段都被设置为x，Linux系统将密码保存在单独的文件（shadow文件，位于/etc/shadow）

7.1.2 /etc/shadow

shadow文件有9个字段，

  1. 登录名

  2. 加密后的密码

  3. 自上次修改密码后已经过去的天数（从1970年1月1日开始计算）

  4. 多少天后才能修改密码

  5. 多少天后必须修改密码

  6. 密码过期前多少天提醒用户更改密码

  7. 密码过期后多少天禁用用户账户

  8. 用户账户被禁用的日期（以从1970年1月1日到当时的天数表示）

  9. 预留给以后的字段

7.1.3 添加新用户-useradd

useradd默认值文件位于/etc/default/useradd，安全设置文件/etc/login.defs；需要root账户登录或者通过sudo运行。

useradd命令选项

使用- D修改默认值选项表

7.1.4 删除用户 userdel

默认情况下userdel命令只删除/etc/passwd和/etc/shadow文件的用户信息，属于该账户的文件会被保留，**-r**选项删除用户的$HOME目录及邮件目录，系统中仍可能存在已删除用户的其他文件。

7.1.5 修改用户

  • usermod：修改用户账户字段，指定组以及辅助组
  • passwd：修改已有账户密码
  • chapasswd：从文件中读取登录名及密码并更新密码
  • chage：修改密码的过期日期
  • chfn：修改账户的备注信息
  • chsh：修改账户的默认登录shell

1. usermod

大部分选项与useradd一样，

  • -l：修改用户账户的登录名
  • -L：锁定账户，禁止用户登录；修改了shadow文件的密码字段，以特殊符号（一般是！号）代替加密后的密码
  • -p：修改账户密码
  • -U：解除锁定，恢复用户登录

2. passwd 和 chpasswd

系统中任何用户都能修改自己的密码，只有root用户有权限修改别人的密码。

只是用passwd修改自己的密码，passwd test修改test用户密码。-e选项强制用户下次登录时修改密码。

chpasswd：大量用户修改密码。从标准输入自动读取一系列以冒号分隔的登录名和密码对偶（username:password），自动对密码加密。

7.2 使用Linux组

组权限允许多个用户对系统对象（比如文件、目录或设备等）共享一组权限

每个组都有一个唯一的组id（GID）和组名

7.2.1 /etc/group文件

有4个字段；系统账户组分配的GID低于500，普通用户组的GID从500开始分配；

group文件是标准的文本文件，可以手动编辑。

  1. 组名
  2. 组密码
  3. GID
  4. 属于该组的用户列表；当用户在/etc/passwd文件中指定某个组为主要组时，该用户不会再作为组成员出现在/etc/group文件中。

7.3 理解文件权限

7.3.1 使用文件权限符号

第一个字段表示对象的类型：

  • -：代表文件
  • d：代表目录
  • l：代表链接
  • c：代表字符设备
  • b：代表块设备
  • p：代表具名管道
  • s：代表网络套接字

3组权限的对象：

  1. 对象的属主的权限
  2. 对象的属组的权限
  3. 系统其他用户的权限

3种权限：

  • r：代表对象是可读的，值为4
  • w：代表对象是可写的，值为2
  • x：代表对象是可执行的，值为1

7.3.2 默认文件权限

对文件而言，全权限值是666；对目录而言，全权限值是777；

umask值是掩码，屏蔽掉不想授权该安全级别的权限。

7.4 更改安全设置

7.4.1 修改权限

  • u：代表用户；属主
  • g：代表组；属组
  • o：代表其他用户
  • a：代表上述所有
  • +：增加权限
  • -：移除权限
  • =：设置权限

权限：

  • s：SUID或者SGID，临时拥有文件属主的执行权限

7.4.2 改变从属关系

chown：改变文件的属主和属组；root用户才能修改属主；

chgrp：改变文件的默认属组

7.5 共享文件

Linux共享文件的方法是创建组。

每个文件和目录有3个额外的信息位

  • SUID：当用户执行该文件时，程序以文件属主的权限运行
  • SGID：
    • 对文件而言，程序以文件属组的权限运行
    • 对目录而言，该目录中创建的新文件会以目录的属组作为默认组
  • 粘滞位：应用于目录时，只有文件属主可以删除或重命名该目录中的文件

7.6 访问控制列表

更先进的文件和目录安全方法：访问控制列表（access control list，ACL）

ACL允许指定包含多个用户或组的列表以及为其分配的权限。

setfacl：为用户或组分配权限

getfacl：查看分配给文件或目录的ACL

第8章 管理文件系统

8.1 探索Linux文件系统

8.1.1 Linux文件系统的演进

  1. ext文件系统

扩展文件系统（extended filesystem，简称ext），使用i节点（inode）跟踪文件信息。在每个物理存储设备创建一个单独的表（i节点表）保存文件信息（文件名、文件大小、文件属主、文件属组、文件访问权限、只想存有文件数据的每个块的指针），文件系统通过i节点号表示文件。缺点：文件大小不超过2GB。

  1. ext2

升级版ext，i节点表新增文件的创建时间、修改时间、以及最后一次访问时间；调整最大文件大小

8.2.1 创建分区

  • fdisk

硬盘设备名称格式：

  • SATA/SCSI驱动器：设备命名格式为/dev/sdx，其中x表示检测到的顺序（abc表示）。
  • SSD NVM驱动器：/dev/nvme Nn#，N是序号（从0开始）#是名称空间编号（从1开始）
  • IDE驱动器：/dev/hdx，其中x表示检测到的顺序（abc表示）。

10.1 vim编辑器

vim -r file_name：恢复.swp文件

命令模式：

  • i：切换到输入模式，从光标当前位置开始输入

  • a：切换到输入模式，从光标下一个位置开始输入

  • o（小写）：切换到输入模式，在当前行下方插入一个新行，开始输入

  • O（大写）：切换到输入模式，在当前行上方插入一个新行，开始输入

  • [n]dd：n为数字，光标向下剪切n行

  • [n]yy：n为数字，光标向下复制n行

  • p（小写）：粘贴剪切板内容到光标下方

  • P（大写）：粘贴剪切板内容到光标上方

  • n<space>：n为数字，光标向右移动n个字符

  • n<enter>：n为数字，光标向下移动n行

  • gg：移动到第一行

  • G：移动到第最后一行

  • nG：，n为数字，移动到第n行

  • /word：向下搜索“word”字符串，n匹配下一个，N匹配上一个

  • ?word：向上搜索“word”字符串，n匹配下一个，N匹配上一个

  • :s/old/new/g：替换当前行内所有字符串”old”为”new”

  • :n,ms/old/new/g：替换第n到第m行所有字符串”old”为”new”

  • :%s/old/new/g：替换文件中所有字符串”old”为”new”

  • :%s/old/new/gc：替换文件中所有字符串”old”为”new”，每次替换时提醒。
