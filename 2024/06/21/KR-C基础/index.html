fatal error C1010: unexpected end of file while looking for precompiled header directive

快捷键：

  1. F1：光标移动到要查看的函数按F1查看详细内容

  4. F9：设置断点

  8. Ctrl + F10：执行到光标

在调试模式下设置窗口：View –> DebugWindows 选择要显示的窗口

  1. memory：查看内存
  2. watch：查看变量的值
  3. stack：查看堆栈
  4. register：寄存器窗口

switch…case的原理就是做了一个有序表，称为case address table即case地址表，存储相应case值的地址。存储的地址顺序按照case值排序，不是按照代码先后顺序排序。case值一般为连续的值，若中间空了，就填上default代码的地址值，没有default的话，就是switch...case的下一条语句 system("pause");的地址。有break就执行跳转的汇编指令。没有break就会执行到下一个case的代码。

case 0就是地址0x0040D7AC的第0个字节，case 1就是第1个字节。相应的字节存储要执行代码的地址值。case 0在地址0x0040D7AC的第0个字节存储的值是0x0040D725也就是printf("n == 0\r\n");代码所在的地址。

基于以上原理，switch...case不能进行区间的比较 a < x < b，因为swtich...case本质上就是查表，也不能case小数。case不能是变量。

递归和循环的优缺点：

  1. 递归擅长解决非线性问题，循环擅长解决线性问题。线性：数据关系有唯一的前驱，唯一的后继，n的前驱是n-1,n的后继是n+1。
  2. 递归效率低，递归的斐波那契数列特别慢。

06、函数的机制

栈（stack）：后进先出的数据结构，允许插入和删除数据的一端（活动的一端；低地址的一端）称为栈顶（top），另一端（固定的一端；高地址的一端）则称为栈底（bottom）。保存函数的返回地址。

入栈：栈顶做减法；

出栈：栈顶做加法；

调用约定使用的两种方法：

  1. int __cdecl GetFib(int n, int m, int y)
  2. Project –> Settings –> C/C++ –> 分类 –>Code Generation –> Calling converesion

四大分区：code、data、stack、heap（堆）

data（全局变量、静态变量、常量）：Inited（readonly（常量）、read and write（可读写变量））、Uninit

stack：局部变量和参数变量
