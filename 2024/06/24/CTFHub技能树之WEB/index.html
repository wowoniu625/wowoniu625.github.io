信息泄露

Git泄露

当开发人员使用git进行版本控制，对站点进行自动部署。如果配置不当，可能将.git文件夹直接部署到线上环境，这就引起了git泄露漏洞。

使用到的工具：BugScanTeam的GitHack

Usage：

Log

使用dirsearch进行目录扫描，发现有git泄露

使用BugScanTeam的GitHack工具获取.git文件夹，注意使用python2

进入到文件夹中，使用git log查看提交记录，

使用git show查看flag

Stash

git stash命令用于暂时保存没有提交的工作。运行该命令后，所有没有commit的代码，都会暂时从工作区移除，回到上次commit时的状态。

使用BugScanTeam的GitHack工具获取.git文件夹，注意使用python2

在文件夹中使用命令git stash list列出所有暂时保存的工作

使用git stash pop取出最近一次的修改。或者使用git stash apply stash@{0} 指定取出某一次修改。

恢复了一个文件，flag在文件中

Index

使用GitHack下载.git文件夹

直接在项目文件夹中就看到TXT文件

00截断

00截断原理：00截断漏洞是一种服务器解析漏洞，存在于Web服务器或应用程序中的文件路径处理功能中。是利用了某些解析函数在处理文件路径时，会将输入的文件名字符串以空字符（\0）截断为文件名和扩展名两部分。

php环境00截断条件：

  • php版本小于5.3.29
  • magic_quotes_gpc = Off

上传一个木马<?php @eval($_POST['cmd']);?>，查看网页源代码，看到题目给的注释提示

代码解读：

  • $_FILES['file']['name']表示上传文件的原始名称。
  • basename()函数获取上传文件的基本名称，并赋值给$name。
  • pathinfo()函数返回一个关联数组，有三个值：dirname，basename,extention。将extention赋值给$ext
  • 白名单判断后缀名。
  • road接受一个GET传参，上传一个目标路径。目标路径拼接随机数，日期和$ext后缀名。
  • move_uploaded_file()函数将上传的文件从临时目录移动到目标路径

思路：

  • 修改文件后缀名，使符合白名单。
  • 修改上传路径，因为无回显上传成功后的文件名称是什么，所以使用%00截断随机的数字和时间函数。move_uploaded_file 函数，在读取$des是读到%00发生了截断，导致存入路径就到%00之前。
  • 然后使用蚁剑连接/upload/muma.php
