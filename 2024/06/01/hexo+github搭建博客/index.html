hexo+github博客搭建

发表于2024-06-01|更新于2024-06-01
|阅读量:

环境准备

通过nvs安装node.js

通过nvs安装node.js。nvs能在不同的node.js版本之间快速切换

在cmd中输入winget install jasongin.nvs会将nvs默认安装在C盘的。

安装在D盘的方法：

  1. 在github的release中下载nvs-1.7.1.msi文件，双击安装。

​

  1. 选择安装路径。

  1. 验证安装：在cmd中输入nvs -v，

使用方法：

  1. 添加最新的node.js$ nvs add latest，

​ 遇到权限问题，

解决方法：选择其中一种即可

​ ①、根据提示以管理员权限打开cmd。

​ ②、右击nvs的安装文件夹设置权限为普通用户可以完全控制。

  1. 查看下载的node.js版本nvs ls

  2. 临时使用node.js，关闭cmd窗口后失效。nvs use node/22.2.0/x64

  3. 永久使用 node/22.2.0/x64，即在任意cmd窗口打开都用使用。 nvs link node/22.2.0/x64。报错了，不会解决。直接手动添加PATH。

​

  1. 在cmd窗口中输入node -v会输出版本，node.js就安装成功，并且可以全局访问。

安装git

在[git](Git - Downloading Package (git-scm.com))官网中下载合适的版本进行安装。下载后双击运行安装。

安装教程参考：Git 详细安装教程（详解 Git 安装过程的每一个步骤）_git安装-CSDN博客

安装完 Git 之后，要做的第一件事就是设置你的用户名和邮件地址。 这一点很重要，因为每一个 Git 提交都会使用这些信息，它们会写入到你的每一次提交中，不可更改：

再次强调，如果使用了 --global 选项，那么该命令只需要运行一次，因为之后无论你在该系统上做任何事情， Git 都会使用那些信息。 当你想针对特定项目使用不同的用户名称与邮件地址时，可以在那个项目目录下运行没有 --global 选项的命令来配置。

安装hexo框架

NPM（Node Package Manager）是一个为Node.js JavaScript运行环境设计的包管理工具。它使开发者能够轻松地下载、安装、更新和管理JavaScript库和框架。

使用npm安装hexo。npm install [-g] hexo ，可选项-g代表全局安装，将安装的包下载到nvs\node\22.2.0\x64\node_modules文件夹中。局部安装就是将安装的包下载到运行当前命令的目录中。

搭建博客

本地搭建hexo博客

在博客的文件夹中进行如下操作

  1. 初始化hexohexo init <folder>

  2. 更改工作目录cd <folder>，安装依赖npm install

  3. hexo s就能在本地访问hexo博客了。

部署到GitHub

创建GitHub仓库

注册一个GitHub账号，然后创建一个仓库，仓库名必须为<user>.github.io，

访问就可以看到刚才创建的GitHub page，因为没有index文件，所以显示的是README.md的内容。

配置SSH密钥

右击打开Git Bash输入命令：ssh-keygen -t rsa -C "user_email"#邮箱是GitHub的邮箱-t选择密钥类型（此处为rsa），-C添加注释。将生成的文件的公钥复制到GitHub–>setting–> SSH and GPG keys中。点击新建SSH-KEY

安装插件

npm install hexo-deployer-git --save

配置站点根目录下_config.yml文件

上传

hexo d将hexo根目录下的public文件的里的文件上传到GitHub

博客美化

更换主题：hexo-theme-butterfly

jerryc127/hexo-theme-butterfly: 🦋 Hexo 主题：蝴蝶 — jerryc127/hexo-theme-butterfly: 🦋 A Hexo Theme: Butterfly (github.com)

下载主题到/thems/目录下

git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

修改 Hexo 根目录下的 _config.yml，把主题改为 butterfly

下载安装pug 以及 stylus 的渲染器npm install hexo-renderer-pug hexo-renderer-stylus --save

将主题的配置文件/thems/butterfly/_config.yml复制到hexo博客根目录下，改名为_config.butterfly.yml。这样子只需要在hexo的根目录下配置主题就可以了。

BUG解决

typora和hexo图片路径不一致，在网页上无法显示

typora设置

下载插件

npm install hexo-image-link --save

md文件图片格式

./post_name/img_id
